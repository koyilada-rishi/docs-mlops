Observability — Prometheus, Grafana, Evidently

Knowing when things break before users do. Three layers of observability: infrastructure metrics (Prometheus), business/ML metrics (Grafana dashboards), and data/model drift (Evidently). The feedback loop that triggers retraining automatically.

Three layers of observability
Layer 1 — Infrastructure
K3s pod health, CPU/memory, request rate, error rate, latency p50/p99. Tells you: is the service running? Prometheus + kube-state-metrics + node_exporter.
Layer 2 — ML metrics
Prediction confidence distribution, model version in service, inference latency, batch sizes. Tells you: is the model behaving normally? Custom Prometheus metrics from FastAPI /metrics.
Layer 3 — Data/model drift
Feature distribution shift, prediction drift, data quality degradation. Tells you: is the world changing in ways the model wasn't trained for? Evidently reports from MinIO inference logs.

Drift types and detection
Data drift (input features)
The distribution of input features shifts from what the model was trained on. Example: average MedInc in requests drops from 3.87 to 2.10 as the user base shifts to lower-income regions. Model predictions are now systematically biased.
PSI testKS testneeds no labels
Concept drift (label relationship)
The relationship between features and target changes. Example: house prices in a neighbourhood spike due to new metro line — same features, different price. Model keeps predicting old prices. Hardest to detect — needs ground truth labels from production.
needs labelshardest to catch
Prediction drift (output)
The distribution of model outputs shifts without any change to the model. Easier to detect — no labels needed. Serves as a proxy for both data drift and concept drift. Monitor this first.
no labels neededearly signal
Statistical tests used by Evidently
PSI (Population Stability Index)<0.1 = stable, 0.1–0.2 = monitor, >0.2 = alert + retrain. Works for numeric and categorical features.
KS test (Kolmogorov-Smirnov)Compares two continuous distributions. p-value <0.05 = drift detected. Good for numeric features.
Chi-squared testFor categorical features. Compares frequency tables between reference and current window.
Jensen-Shannon divergenceSymmetric KL divergence. Works on any distribution. Good for prediction drift and output monitoring.
Wasserstein distanceEarth mover's distance. Sensitive to small shifts in heavy-tailed distributions. Used for income/price features.
Evidently — drift monitoring in practice
Evidently CronJob — runs nightly on K3s
from evidently.report import Report from evidently.metric_preset import DataDriftPreset, DataQualityPreset from evidently.metrics import ColumnDriftMetric import pandas as pd, boto3, json from datetime import date s3 = boto3.client("s3", endpoint_url="http://192.168.1.15:30900", aws_access_key_id="minioadmin", aws_secret_access_key="minioadmin") # Reference: training data (the ground truth) reference = pd.read_parquet("data/processed/features.parquet") # Current: last 7 days of inference logs from MinIO logs = [] resp = s3.list_objects_v2(Bucket="mlops-artifacts", Prefix="inference-logs/") for obj in resp.get("Contents", []): body = s3.get_object(Bucket="mlops-artifacts", Key=obj["Key"]) logs.append(json.loads(body["Body"].read())) current = pd.DataFrame([l["input"] for l in logs]) # Run Evidently report report = Report(metrics=[ DataDriftPreset(), # PSI + KS for all features DataQualityPreset(), # null rates, value ranges ColumnDriftMetric("MedInc"), # per-feature drill-down ]) report.run(reference_data=reference, current_data=current) # Save HTML report to MinIO report.save_html("/tmp/drift_report.html") s3.upload_file("/tmp/drift_report.html", "mlops-artifacts", f"evidently/reports/{date.today()}.html") # Extract drift score → push to Prometheus pushgateway result = report.as_dict() drift_score = result["metrics"][0]["result"]["dataset_drift"] if drift_score: print("DRIFT DETECTED — triggering retraining workflow") # POST to GitHub Actions workflow_dispatch
Grafana dashboards
What each dashboard panel shows
Request rateRequests per second by model_version. Spot canary vs stable traffic split.
Latency heatmapp50, p95, p99 latency over 24h. Alert if p99 > 200ms.
Error rateHTTP 4xx, 5xx per minute. Alert if error_rate > 1%.
Prediction distributionHistogram of model outputs over time. Visual drift detection before Evidently fires.
Null fill rateHow many inputs needed imputation. Rising rate = upstream data quality degrading.
PSI trendEvidently drift scores per feature over time. Alert if PSI > 0.2.
Pod healthReplica count, CPU %, memory %, restart count for all mlops namespace pods.
MinIO storageBucket sizes over time. Alert before disk fills up.
