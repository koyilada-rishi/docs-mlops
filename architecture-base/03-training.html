Registry: Production
Model testing happens before the MinIO push. Only models that pass the evaluation gate get promoted. Failed models never reach the registry.

Evaluation gates — the two-stage check
The SRE analogy: validation set = staging environment. Test set = production gate. You tune on staging. You release only if production gate passes. You never deploy straight from development to production.
Validation set (staging)
Used to tune hyperparameters. Compare model versions. Select best architecture. The model never trains on this. But you look at it repeatedly, so it has indirect influence on model selection.
~15% of datacan look multiple times
Test set (production gate)
Locked away. Touched exactly once — final evaluation before promotion. If you look at it more than once, it becomes a second validation set and your evaluation is optimistic.
~15% of dataone-time only
Evaluation gate threshold
Define the minimum bar before any training run begins. If test_rmse > threshold, the model is not promoted. It stays in Staging in the registry. CI sends an alert.
THRESHOLD = { "test_rmse": 0.50, "val_rmse": 0.52, "min_r2": 0.80, } if test_rmse < THRESHOLD["test_rmse"]: promote_to_production(run_id)
Hyperparameter tuning with Optuna
Optuna integration with MLflow — log every trial
import optuna, mlflow from sklearn.ensemble import RandomForestRegressor from sklearn.metrics import mean_squared_error def objective(trial): params = { "n_estimators": trial.suggest_int("n_estimators", 50, 500), "max_depth": trial.suggest_int("max_depth", 3, 20), "min_samples_leaf": trial.suggest_int("min_samples_leaf", 1, 20), "max_features": trial.suggest_categorical( "max_features", ["sqrt", "log2", 0.5]), } with mlflow.start_run(nested=True): # nested run per trial mlflow.log_params(params) rf = RandomForestRegressor(**params, random_state=42, n_jobs=-1) pipe = build_pipeline(rf) pipe.fit(X_train, y_train) val_rmse = mean_squared_error(y_val, pipe.predict(X_val)) ** 0.5 mlflow.log_metric("val_rmse", val_rmse) return val_rmse # Optuna minimises this with mlflow.start_run(run_name="optuna-sweep-v1"): study = optuna.create_study(direction="minimize") study.optimize(objective, n_trials=50, n_jobs=4) best = study.best_params mlflow.log_params({f"best_{k}": v for k, v in best.items()})

Complete training script
train.py — production-ready end-to-end
import pandas as pd, numpy as np import mlflow, mlflow.sklearn import subprocess, json from sklearn.pipeline import Pipeline from sklearn.compose import ColumnTransformer from sklearn.impute import SimpleImputer from sklearn.preprocessing import StandardScaler, OneHotEncoder from sklearn.ensemble import RandomForestRegressor from sklearn.metrics import mean_squared_error, r2_score # --- Config --- MLFLOW_URI = "http://192.168.1.15:30500" EXPERIMENT = "california-housing" RMSE_THRESHOLD = 0.50 MODEL_NAME = "california-housing" # --- Load data (from DVC-tracked local files) --- X_train = pd.read_parquet("data/splits/X_train.parquet") X_val = pd.read_parquet("data/splits/X_val.parquet") X_test = pd.read_parquet("data/splits/X_test.parquet") y_train = pd.read_parquet("data/splits/y_train.parquet").squeeze() y_val = pd.read_parquet("data/splits/y_val.parquet").squeeze() y_test = pd.read_parquet("data/splits/y_test.parquet").squeeze() # --- Build pipeline --- num_cols = ["MedInc", "HouseAge", "AveRooms", "AveOccup"] cat_cols = ["OceanProximity"] pre = ColumnTransformer([ ("num", Pipeline([ ("imp", SimpleImputer(strategy="median")), ("sc", StandardScaler()), ]), num_cols), ("cat", Pipeline([ ("imp", SimpleImputer(strategy="most_frequent")), ("ohe", OneHotEncoder(handle_unknown="ignore")), ]), cat_cols), ]) pipe = Pipeline([ ("pre", pre), ("model", RandomForestRegressor(n_estimators=100, random_state=42, n_jobs=-1)), ]) # --- Track with MLflow --- mlflow.set_tracking_uri(MLFLOW_URI) mlflow.set_experiment(EXPERIMENT) with mlflow.start_run(run_name="rf-v1-baseline") as run: pipe.fit(X_train, y_train) val_rmse = mean_squared_error(y_val, pipe.predict(X_val)) ** 0.5 test_rmse = mean_squared_error(y_test, pipe.predict(X_test)) ** 0.5 val_r2 = r2_score(y_val, pipe.predict(X_val)) mlflow.log_params({ "n_estimators": 100, "n_train_rows": len(X_train), "dvc_hash": subprocess.check_output( ["git", "rev-parse", "HEAD"]).decode().strip(), }) mlflow.log_metrics({ "val_rmse": round(val_rmse, 4), "test_rmse": round(test_rmse, 4), "val_r2": round(val_r2, 4), }) # Save imputer/scaler stats for audit imp_stats = dict(zip(num_cols, pipe.named_steps["pre"] .named_transformers_["num"] .named_steps["imp"].statistics_.tolist())) mlflow.log_dict(imp_stats, "imputer_statistics.json") # Log model — ONLY if passes evaluation gate if test_rmse < RMSE_THRESHOLD: mlflow.sklearn.log_model(pipe, "model", registered_model_name=MODEL_NAME) print(f"✓ Model promoted. RMSE={test_rmse:.4f}") else: print(f"✗ Model NOT promoted. RMSE={test_rmse:.4f} > {RMSE_THRESHOLD}") exit(1) # fail the CI step
Common training failures
Data leakageTarget in features, scaler fit on full dataset before split, or rolling average includes current row. Silent — model accuracy looks great, production fails.
Wrong split typeShuffling time-series data. Train sees future events. Fix: temporal split — train on past, val/test on future. Never shuffle.
OverfittingLow train error, high val error. Model memorised training data. Fix: regularisation, smaller model, more data, cross-validation.
MLflow tracking URI wrongRuns logged to local ./mlruns instead of the K3s tracking server. Always set_tracking_uri() explicitly at script start.
Missing DVC datadvc pull not run → local data is stale or missing → training on wrong version. Fix: always dvc pull as first step in training script.
