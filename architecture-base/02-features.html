Stage 02

Feature Engineering — Raw data to model language

A model cannot learn from raw strings, nulls, or timestamps. Feature engineering translates real-world data into a fixed-width numeric matrix — the only language a model speaks. This stage has more impact on model performance than the algorithm choice.

Why it matters more than the algorithm
The 80/20 rule
80% of model performance gain comes from better features, not a better algorithm. Switching from RandomForest to XGBoost: +2% RMSE improvement. Adding a rolling 7-day average: +15% improvement.
What goes wrong without it
Model sees "Mumbai" as a string — can't process it. Null → crash or silent zero. "2019-08-14" as a string → misses day_of_week, hour, seasonality entirely. The model is blind to structure in the data.
The model's contract
Every model expects a fixed-width numeric matrix with the same number of columns in the same order with no nulls — every single call. Feature engineering enforces this contract at both training and serving time.
Numeric transformations
Standardisation vs min-max — when to use which
StandardScalerMean=0, Std=1. For SVM, KNN, PCA, linear regression, neural networks. Algorithms that use distance or gradient descent.
MinMaxScalerRange [0,1]. For neural networks, images, when you know absolute bounds. Sensitive to outliers.
RobustScalerUses median + IQR. Robust to outliers. Good for skewed distributions with many extreme values.
No scalingTree models (RandomForest, XGBoost) don't need scaling — they use thresholds, not distances.
Log transform + outlier clipping
import numpy as np # Log transform for right-skewed data (price, income, counts) df["log_price"] = np.log1p(df["price"]) # log(1+x) — safe for 0 # Clip outliers to training percentiles p1 = X_train["income"].quantile(0.01) p99 = X_train["income"].quantile(0.99) X_train["income"] = X_train["income"].clip(p1, p99) X_val["income"] = X_val["income"].clip(p1, p99) # At serve time: use training p1/p99, log clip events

Temporal features
Point-in-time + lag + rolling — complete example
import pandas as pd, numpy as np df["ts"] = pd.to_datetime(df["event_ts"]) # USE EVENT_TS, not ingest_ts # Point-in-time features df["hour"] = df["ts"].dt.hour df["day_of_week"] = df["ts"].dt.dayofweek df["is_weekend"] = df["ts"].dt.dayofweek >= 5 df["month"] = df["ts"].dt.month # Cyclical encoding — avoids hour-23 to hour-0 jump df["hour_sin"] = np.sin(2 * np.pi * df["hour"] / 24) df["hour_cos"] = np.cos(2 * np.pi * df["hour"] / 24) # Recency df = df.sort_values("ts") df["days_since_last_purchase"] = ( df["ts"] - df.groupby("user_id")["ts"].shift(1) ).dt.days # Rolling average — shift(1) FIRST to exclude current row! df["sales_7d_mean"] = df["sales"].shift(1).rolling(7).mean() df["sales_7d_std"] = df["sales"].shift(1).rolling(7).std()
Common mistake: Rolling average without shift(1). If the current row's sales value is inside the 7-day window you're averaging, you're predicting today's sales using today's sales — that's leakage. Always shift first.
The sklearn Pipeline — preventing train-serve skew
Complete production pipeline — numeric + categorical + model
from sklearn.pipeline import Pipeline from sklearn.compose import ColumnTransformer from sklearn.impute import SimpleImputer from sklearn.preprocessing import StandardScaler, OneHotEncoder from sklearn.ensemble import RandomForestRegressor import mlflow num_cols = ["MedInc", "HouseAge", "AveRooms", "AveOccup"] cat_cols = ["OceanProximity"] num_pipe = Pipeline([ ("impute", SimpleImputer(strategy="median")), ("scale", StandardScaler()), ]) cat_pipe = Pipeline([ ("impute", SimpleImputer(strategy="most_frequent")), ("encode", OneHotEncoder(handle_unknown="ignore")), ]) pre = ColumnTransformer([ ("num", num_pipe, num_cols), ("cat", cat_pipe, cat_cols), ]) full_pipe = Pipeline([ ("pre", pre), ("model", RandomForestRegressor(n_estimators=100, random_state=42)), ]) full_pipe.fit(X_train, y_train) # Fit on training data ONLY with mlflow.start_run(): mlflow.sklearn.log_model(full_pipe, "model") # Frozen inside the pkl: # imputer.statistics_ — median per col from X_train # scaler.mean_ and scale_ — from imputed X_train # ohe.categories_ — all cats seen in training # Serving: full_pipe.predict(X_new) — no re-fitting. Ever.
Feature selection
Tree importance + SHAP
import shap, pandas as pd # Tree-based importance (free from RandomForest) importances = pd.Series( model.feature_importances_, index=feature_names ).sort_values(ascending=False) keep = importances[importances > 0.01].index # SHAP — more reliable, shows direction of impact explainer = shap.TreeExplainer(model) shap_values = explainer.shap_values(X_val) mean_shap = pd.Series( abs(shap_values).mean(axis=0), index=feature_names ).sort_values(ascending=False) # Always: fit selector on training data only
What to always drop
Zero varianceSame value in every row — contributes nothing
>90% nullsImputation will dominate — not real signal
Duplicate columnsCorrelation = 1.0 — one is redundant
Raw ID columnsuser_id=12345 has no predictive value as a number
Raw timestampsExtract temporal features instead — never feed raw datetime
