Storage layer — what lives in each MinIO bucket
📦 mlops-raw/
Raw landing zone. All ingestion writes here first. Human-readable path keys (source=db/date=2025-08-01/). 90-day S3 lifecycle expiry. Immutable audit record.
ETL writesS3 lifecycle 90dParquetSource partitioned
🗄️ mlops-dvc-cache/
DVC remote. Content-addressed by MD5 hash. Permanent — never deleted. dvc push writes here, dvc pull reads from here. Deduplication: same file = same hash = stored once.
DVC push/pullMD5 keyedPermanentDeduped
🤖 mlops-models/
MLflow artifact store. Frozen sklearn Pipeline pkl (imputer stats, scaler means, OHE categories all locked). KServe loads model via storageUri at deploy time.
MLflow artifactspipeline.pklconda.yamlKServe storageUri
📊 mlops-artifacts/
Evidently HTML drift reports, SHAP plots, inference request logs (one JSON per request). Evidently's CronJob reads inference-logs/ to build drift report.

Orchestration — who schedules and runs every stage
🌀 Apache Airflow
DAG-based scheduler. 3 production DAGs: ingestion (daily 1AM cron), training (triggered after ingest via TriggerDagRunOperator), drift-retrain (nightly after Evidently via BranchPythonOperator checks PSI). KubernetesPodOperator runs heavy steps as isolated K3s pods.
DAGPythonOperatorBashOperator KubernetesPodOperatorS3KeySensor BranchPythonOperatorTriggerDagRunOperator XComKubernetesExecutor Airflow ConnectionsVariables

⚙️ Kubeflow Pipelines (KFP)
ML-native orchestration. Each step is a typed container with Dataset/Model/Metrics artifacts tracked via MLMD for automatic lineage. dsl.If enables conditional promotion gates. Katib runs parallel HPO trials as separate K8s pods using Bayesian search — smarter than grid.
@dsl.componentContainerSpec dsl.If (conditional)Katib HPO MLMD lineageBayesian search pipeline.yamlArgo Workflows Dataset / Model artifacts

🔁 CI/CD — GitHub Actions
Git push → CI: ruff lint + pytest + dvc repro + test_skew.py + Docker build + GHCR push. Merge → CD: kubectl apply + rollout status + smoke_test.py (latency + schema + positive prediction). Drift → workflow_dispatch retrain. Rollback: git revert + MLflow version transition.

Supporting tools — complete the production stack
✅ Great Expectations
Data quality as code. Expectation Suites run at every ingest. Checkpoints halt pipeline on fail. Data Docs (HTML report) auto-saved to MinIO. GE vs Pandera: use Pandera for fast in-code, GE for full audit platform.
Expectation SuiteCheckpointData DocsMinIO backend

🏪 Feast (Feature Store)
Offline store (MinIO Parquet) gives point-in-time correct features for training. Online store (Redis) gives sub-millisecond features at serving time. Materialise on a schedule to keep Redis fresh.
FeatureViewRedis onlinePoint-in-timeMaterialise
Study Feast →
🔷 dbt (data build tool)
SQL feature engineering in your warehouse. ref() builds DAG automatically. Incremental models process only new rows. Rolling windows, group-bys, joins — all in pure SQL. dbt test asserts data quality after each model run.
dbt modelsref()Incrementaldbt testdbt docs
Study dbt →
📦 BentoML
Package model + code + deps into a "Bento". bentoml build + bentoml containerize → Docker image in two commands. Async Runner handles concurrent inference. Faster to start with than KServe. Deploy anywhere.
BentoRunnerbentoml servecontainerize
