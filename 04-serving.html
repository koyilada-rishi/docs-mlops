Serving Pipeline — FastAPI, KServe, Canary

How a trained model becomes an API that other services can call. FastAPI gateway handles auth and validation, KServe InferenceService handles model execution on K3s, canary rollout reduces risk, and every request is logged to MinIO.

FastAPI gateway
What the gateway does
JWT authValidates Bearer token. Rejects unauthenticated requests with 401.
Pydantic validationEnforces input schema. Returns 422 on malformed input — never passes garbage to model.
Rate limiting100 requests/minute per API key. Returns 429 with Retry-After header.
PreprocessingApplies feature column ordering before forwarding to KServe. No transform logic — that's in the pipeline pkl.
Request loggingEvery request + response + latency logged to MinIO as JSON. Used by Evidently for drift monitoring.
Health checkGET /health → 200. Used by K3s liveness probe.
serve.py — complete FastAPI endpoint
import mlflow, boto3, json, time, uuid from fastapi import FastAPI, Depends, HTTPException from fastapi.security import HTTPBearer from pydantic import BaseModel from typing import Optional from datetime import datetime app = FastAPI() security = HTTPBearer() mlflow.set_tracking_uri("http://192.168.1.15:30500") pipe = mlflow.sklearn.load_model( "models:/california-housing/Production") FEATURES = ["MedInc","HouseAge","AveRooms", "AveBedrms","Population","AveOccup", "Latitude","Longitude"] s3 = boto3.client("s3", endpoint_url="http://192.168.1.15:30900", aws_access_key_id="minioadmin", aws_secret_access_key="minioadmin") class House(BaseModel): MedInc: Optional[float] = None HouseAge: Optional[float] = None AveRooms: Optional[float] = None AveBedrms: Optional[float] = None Population: Optional[float] = None AveOccup: Optional[float] = None Latitude: Optional[float] = None Longitude: Optional[float] = None @app.get("/health") def health(): return {"status": "ok"} @app.post("/predict") def predict(h: House, creds=Depends(security)): t0 = time.time() rid = str(uuid.uuid4()) row = [[getattr(h, f) for f in FEATURES]] # pipe handles nulls, scale, predict — no re-fitting price = float(pipe.predict(row)[0]) * 100_000 lat = round((time.time() - t0) * 1000, 1) log = {"request_id": rid, "ts": datetime.utcnow().isoformat(), "input": h.dict(), "prediction": round(price, 2), "model_version": "Production", "latency_ms": lat} # Async log to MinIO (fire and forget) s3.put_object(Bucket="mlops-artifacts", Key=f"inference-logs/{rid}.json", Body=json.dumps(log).encode()) return {"request_id": rid, "price_usd": round(price, 2), "model_version": "Production", "latency_ms": lat}
KServe InferenceService
What KServe provides
Model loadingPulls model artifact from MinIO models/ bucket using URI from MLflow registry
V2 protocolStandardised inference protocol (gRPC + REST) used by KServe. Interoperable with any client.
Auto-scalingScale predictor pods from 0 to N based on request rate. Zero-to-hero scaling.
Transformer podPre/post processing before predictor. Custom Python code.
Canary rolloutTraffic splitting between model versions. KServe handles this natively.
Explainer podOptional SHAP/LIME explainability endpoint at /v1/models/x:explain

Canary rollout strategy
Why canary? Offline evaluation (test set RMSE) doesn't fully capture production behaviour. Real traffic has edge cases, distribution shifts, and interaction patterns your test set missed. Canary exposes the new model to 10% of real traffic before you trust it with 100%.
Stage 1: 10% canary
Deploy v2 with 10% traffic. Monitor latency p99 and prediction distribution in Grafana for 24 hours. If both look healthy, proceed.
Stage 2: 50% split
If 10% passes, go to 50/50. True A/B test. Compare business metrics (revenue, click-through, error rate) directly. Run for 48 hours.
Stage 3: 100% promote
If metrics hold, set canaryTrafficPercent=0, archive old version. MLflow registry: old version → Archived, new version stays Production.
Online vs batch inference
Online inference (real-time)
Single request → immediate response. Used for: house price estimation, fraud detection at checkout, recommendation at page load.

Latency target: p99 < 200ms. Model must be in memory. Cold start (first request after scale-from-zero) may be slow — mitigate with min_replicas=1.
Batch inference (asynchronous)
Whole dataset → results file. Used for: nightly price predictions for all properties, monthly churn scoring for all users.

No latency constraint. Use KServe InferenceGraph or a simple Python loop reading from MinIO and writing results back.
# Batch inference against registered model pipe = mlflow.sklearn.load_model( "models:/california-housing/Production") df = pd.read_parquet("data/raw/all_properties.parquet") df["predicted_price"] = pipe.predict(df[FEATURES]) * 100_000 df.to_parquet("data/predictions/prices.parquet") # Upload predictions to MinIO for downstream use
Handling edge cases at serving time
Null / missing featurePipeline's frozen imputer fills with training median. Never crash, never forward null. Log fill events — high fill rate = drift signal.
Unknown categoryOHE handle_unknown="ignore" → all-zeros row for that category. Model predicts without that feature. Log unseen category — it may mean schema drift.
Out-of-range numericIf MedInc = 99999 (outlier), pipeline clips to p99 from training. Prediction proceeds. Log clip event to MinIO for Evidently drift analysis.
Wrong column orderFEATURES list in serve.py must match training order exactly. Test with test_skew.py in CI — checks that null input equals median-filled input.
Model cold startKServe scales from zero on first request. Model loads from MinIO — takes 2-10 seconds. Set min_replicas=1 for latency-sensitive endpoints.
Pod OOMModel too large for allocated memory limit. Increase limit in manifest or use quantisation (ONNX int8) to shrink model size.
