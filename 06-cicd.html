Stage 06

CI/CD Loop — Git to Deploy to Retrain

Automating the entire MLOps lifecycle. A git push triggers CI (lint, test, dvc repro), CD deploys to K3s, MLflow model promotion triggers a webhook that updates KServe, and detected drift fires the training pipeline again. The loop never stops.

The full loop
git push PR
→
CI: lint+test+dvc
→
Docker build
→
K3s deploy
→
Smoke test
→
MLflow promote

Drift detected

GitHub Actions — CI workflow
.github/workflows/ci.yaml
name: CI Pipeline on: push: branches: [mlops-prep-nayak-main] pull_request: branches: [mlops-prep-nayak-main] env: AWS_ACCESS_KEY_ID: minioadmin AWS_SECRET_ACCESS_KEY: minioadmin jobs: ci: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - name: Set up Python uses: actions/setup-python@v5 with: {python-version: "3.11"} - name: Install dependencies run: pip install -r requirements.txt dvc[s3] - name: Configure DVC remote run: | dvc remote modify minio_remote endpointurl \ http://192.168.1.15:30900 - name: Lint run: | ruff check src/ ruff format --check src/ - name: Unit tests run: pytest tests/unit/ -v - name: Pull data from DVC cache run: dvc pull data/raw/ - name: Run pipeline (only changed stages) run: dvc repro - name: Skew tests run: pytest tests/test_skew.py -v - name: Push pipeline artifacts run: dvc push - name: Build Docker image run: | docker build -t ghcr.io/koyilada-rishi/mlops-serve:${{ github.sha }} . docker push ghcr.io/koyilada-rishi/mlops-serve:${{ github.sha }}
GitHub Actions — CD workflow
.github/workflows/deploy.yaml
name: CD — Deploy to K3s on: workflow_run: workflows: [CI Pipeline] types: [completed] branches: [mlops-prep-nayak-main] jobs: deploy: if: ${{ github.event.workflow_run.conclusion == 'success' }} runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - name: Setup kubectl uses: azure/setup-kubectl@v3 - name: Configure kubeconfig run: | echo "${{ secrets.KUBECONFIG }}" > kubeconfig.yaml export KUBECONFIG=kubeconfig.yaml - name: Update KServe manifest with new image run: | sed -i "s|IMAGE_TAG|${{ github.sha }}|g" \ k8s/inference-service.yaml - name: Apply K3s manifests run: kubectl apply -f k8s/ - name: Wait for rollout run: | kubectl rollout status deployment/mlops-fastapi \ -n mlops --timeout=120s - name: Smoke test run: | python tests/smoke_test.py \ --endpoint http://192.168.1.15:8080/predict \ --max-latency-ms 200
GitHub Actions — retraining trigger
.github/workflows/retrain.yaml — triggered by drift alert
name: Retrain — Drift Detected on: workflow_dispatch: inputs: drift_source: description: "Which feature drifted (e.g. MedInc)" required: true psi_score: description: "PSI score that triggered this" required: true jobs: retrain: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - name: Install run: pip install -r requirements.txt dvc[s3] - name: Log drift event to MLflow run: | python scripts/log_drift_event.py \ --feature ${{ github.event.inputs.drift_source }} \ --psi ${{ github.event.inputs.psi_score }} - name: Pull latest data run: dvc pull data/raw/ - name: Bootstrap new dataset version run: | python scripts/ingest_latest.py dvc add data/raw/ dvc push - name: Run training pipeline run: dvc repro - name: Evaluate and promote if better run: python scripts/promote_if_better.py
Smoke test — validates every deployment
tests/smoke_test.py
import requests, sys, time, argparse parser = argparse.ArgumentParser() parser.add_argument("--endpoint", required=True) parser.add_argument("--max-latency-ms", type=float, default=200) args = parser.parse_args() TEST_CASES = [ {"MedInc": 8.3252, "HouseAge": 41.0, "AveRooms": 6.98, "AveBedrms": 1.02, "Population": 322.0, "AveOccup": 2.55, "Latitude": 37.88, "Longitude": -122.23}, {"MedInc": None, "HouseAge": 28.0, "AveRooms": 5.0, # null test "AveBedrms": 1.0, "Population": 500.0, "AveOccup": 3.0, "Latitude": 34.05, "Longitude": -118.25}, ] for i, tc in enumerate(TEST_CASES): t0 = time.time() resp = requests.post(args.endpoint, json=tc, timeout=10) lat = (time.time() - t0) * 1000 assert resp.status_code == 200, \ f"Test {i}: Expected 200, got {resp.status_code}" assert "price_usd" in resp.json(), \ f"Test {i}: Missing price_usd in response" assert resp.json()["price_usd"] > 0, \ f"Test {i}: Negative price prediction" assert lat < args.max_latency_ms, \ f"Test {i}: Latency {lat:.0f}ms > {args.max_latency_ms}ms" print(f"✓ Test {i}: price=${resp.json()['price_usd']:,.0f}, latency={lat:.0f}ms") print("All smoke tests passed.")
Dockerfile — serving container
FROM python:3.11-slim WORKDIR /app # Install dependencies first (Docker layer cache) COPY requirements.txt . RUN pip install --no-cache-dir -r requirements.txt # Copy application code COPY src/ src/ # Model is NOT baked into the image # It is loaded at runtime from MLflow registry # This keeps the image small and model versioning flexible ENV MLFLOW_TRACKING_URI=http://192.168.1.15:30500 ENV MODEL_NAME=california-housing ENV MODEL_STAGE=Production EXPOSE 8080 CMD ["uvicorn", "src.serve:app", "--host", "0.0.0.0", "--port", "8080", "--workers", "2"]
Rollback strategy
Code rollbackgit revert + re-run CI. Previous Docker image tag still in registry.
Model rollbackMLflow: transition previous version from Archived → Production. CD webhook fires and updates KServe manifest.
KServe rollbackkubectl rollout undo deployment/kserve-controller -n kserve. KServe restores previous InferenceService revision.
Data rollbackgit checkout <sha> → dvc pull → exact previous dataset version available locally.
Config rollbackparams.yaml is in Git. Revert the commit that changed params → dvc repro → previous model results.
